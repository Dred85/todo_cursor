# Tasker

Современное приложение для управления задачами с умным парсингом дат из естественного языка.

## Структура

```
tasker/
├── .cursor/
│   └── rules.md
├── ARCHITECTURE.md
├── PROMPTS.md
├── TASK_PARSING_SPEC.md
├── backend/          # Python + FastAPI
├── webapp/          # NodeJS + TypeScript + React
└── docker-compose.yml
```

## Возможности

- ✨ **Красивый современный UI** с градиентным дизайном
- 📝 **Умный парсинг дат** из естественного языка (русский/английский)
- 📋 **Управление задачами** с фильтрацией по статусу
- ⏰ **Отображение сроков** в удобном формате ("завтра в 18:00", "сегодня в 14:30")
- 🎯 **Статистика** по задачам (всего, к выполнению, выполнено)
- 📱 **Адаптивный дизайн** для мобильных устройств

## Запуск через Docker Compose

Из корня проекта:

```bash
cd tasker
docker compose up --build
```

После запуска:
- **Backend API**: `http://localhost:8000`
  - `GET /api/health` - проверка здоровья сервиса
  - `POST /api/task` - создание задачи
  - `GET /api/tasks?user_id=1` - список задач пользователя
- **WebApp**: `http://localhost:5173` - веб-интерфейс

## Примеры использования

### Создание задачи через UI

Откройте `http://localhost:5173` и введите в поле ввода:
- `Позвонить маме` - задача без даты
- `Купить билеты завтра 18:00` - задача с датой и временем
- `Отправить отчёт в пятницу` - задача с датой (время по умолчанию 09:00)

### Пример запроса к API

```bash
curl -s -X POST http://localhost:8000/api/task \
  -H 'Content-Type: application/json' \
  -d '{"user_id":1,"timezone":"UTC","text":"Купить билеты завтра 18:00","source":"webapp"}'
```

## Устранение проблем

### Ошибка "Ошибка соединения с сервером"

1. **Проверьте, что оба контейнера запущены:**
   ```bash
   docker compose ps
   ```
   Должны быть запущены `tasker_backend` и `tasker_webapp`.

2. **Проверьте логи backend:**
   ```bash
   docker compose logs backend
   ```
   Должны быть сообщения о запуске uvicorn на порту 8000.

3. **Проверьте логи webapp:**
   ```bash
   docker compose logs webapp
   ```
   Должны быть сообщения о запуске Vite dev server.

4. **Проверьте доступность API напрямую:**
   ```bash
   curl http://localhost:8000/api/health
   ```
   Должен вернуться `{"ok":true}`.

5. **Пересоберите контейнеры:**
   ```bash
   docker compose down
   docker compose up --build
   ```

### Проблемы с прокси

Если фронтенд не может подключиться к бэкенду через прокси Vite:
- Убедитесь, что в `vite.config.ts` настроен прокси на `http://backend:8000`
- Проверьте, что контейнеры находятся в одной Docker сети (должно работать автоматически через docker-compose)

### Ошибка "Ошибка создания задачи"

1. **Проверьте логи backend для детальной информации:**
   ```bash
   docker compose logs backend
   ```
   Там будут видны конкретные ошибки валидации или парсинга.

2. **Проверьте консоль браузера (F12):**
   - Откройте Developer Tools → Console
   - Там будут логи ошибок с деталями от backend

3. **Частые причины:**
   - Пустой текст задачи → введите описание
   - Текст слишком короткий (< 2 символов) → увеличьте длину
   - Дата в прошлом → используйте будущую дату
   - Неправильный user_id (<= 0) → используйте положительное число

4. **Проверьте формат запроса:**
   ```bash
   curl -v -X POST http://localhost:8000/api/task \
     -H 'Content-Type: application/json' \
     -d '{"user_id":1,"timezone":"UTC","text":"Позвонить маме","source":"webapp"}'
   ```
